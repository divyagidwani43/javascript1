        Promise:
        What it does: Represents the eventual completion (or failure) of an asynchronous operation and allows chaining .then() or .catch().
        How it works: Promises use the Microtask Queue. They get executed after the current execution context but before the Event Queue.
        Promises go into a VIP queue called the Microtask Queue, which gives them priority over the normal Event Queue (the "waiting" queue, like the one setTimeout uses).
        When a Promise is resolved or rejected, its .then() or .catch() callback gets placed in the Microtask Queue.
        The Microtask Queue has higher priority than the Event Queue (where setTimeout, setInterval, and other events go).
        Promises are executed after the Call Stack is empty but before any other tasks in the Event Queue.
